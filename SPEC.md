# SPEC: Sseclone — AI Video Clipping Tool (Ssemble clone)

This is the single source of truth for WHAT the product does. Every phase prompt
references this file. Do not restate the spec in phase prompts — read it here.

## Product summary

Sseclone turns long-form video (podcasts, streams, webinars, lectures) into
short, social-ready clips automatically. Pipeline:

    upload long video
      → transcribe (word-level timestamps)
      → AI highlight detection (find the most engaging moments)
      → auto-clip (cut short segments around highlights)
      → auto captions (styled, editable subtitles)
      → smart crop (reframe to 9:16 / 1:1 / 16:9 with subject tracking)
      → enhance (B-roll, transitions, sound effects, CTA overlays, templates)
      → export platform-ready MP4s (TikTok, YouTube Shorts, Instagram Reels)

Target users: shorts creators, social media managers, podcasters, agencies.

## Tech stack (fixed — do not substitute)

- Next.js 15+ (App Router) + TypeScript strict mode, single repo
- Node.js worker process (`src/worker/`) for all media processing; started with
  `npm run worker`
- SQLite via better-sqlite3 + Drizzle ORM; migrations in `drizzle/`
- Job queue: SQLite-backed table (`jobs`) polled by the worker. NO Redis, no
  external services, no Docker requirement for dev.
- FFmpeg + ffprobe system binaries invoked via `execa` with explicit arg
  arrays (no shell-string interpolation). All FFmpeg calls live in
  `src/lib/ffmpeg/`.
- Transcription: whisper.cpp binary behind a `Transcriber` interface
  (`src/lib/transcribe/`). Tests use `FakeTranscriber` with fixture
  transcripts — never call real whisper in the default test suite.
- Highlight scoring: local heuristics only (no API keys anywhere):
  audio RMS energy peaks, speech density, keyword/hook phrases, laughter/
  exclamation markers in transcript, scene-change boundaries. Behind a
  `HighlightScorer` interface so an LLM scorer can be added later.
- Smart crop: `SubjectDetector` interface. Default impl: face detection via
  @vladmandic/human (TF.js, CPU) with center-weighted fallback when no face
  is found. Crop keyframes are smoothed (no jitter).
- Captions: word-timed captions stored as JSON; rendered to `.ass` with style
  presets; burned in via FFmpeg `subtitles`/`ass` filter.
- UI: Tailwind CSS. Video preview via native `<video>` element.
- Tests: vitest (unit + integration), Playwright (e2e). Test media is
  GENERATED by `scripts/make-fixtures.sh` (ffmpeg testsrc/sine) — never
  commit binary video files.

## Data model (minimum)

- `projects` — id, name, source video path, status, platform preset, created_at
- `assets` — uploaded/imported media (source videos, B-roll, sound effects)
- `jobs` — id, project_id, type, status (queued|running|done|failed), progress
  0–100, error, payload JSON
- `transcripts` — project_id, segments JSON (word-level: text, start, end)
- `clips` — id, project_id, source in/out points, score, title, status
- `clip_edits` — per-clip edit state JSON: caption style + edited words,
  crop keyframes, aspect ratio, timeline (segments, B-roll slots, transitions,
  SFX, CTA overlays), template id
- `exports` — clip_id, preset, output path, status

## Feature checklist (full clone)

1. **Import**: upload video file (mp4/mov/webm, ≤2 GB); probe metadata
   (duration, resolution, fps, audio streams); reject non-video with a clear
   error.
2. **Auto transcription**: word-level timestamps; stored; viewable.
3. **AI highlight detection + auto clipping**: given a long video, produce N
   ranked candidate clips (default 5–10, configurable 15–90 s each), with
   scores and reasons ("high energy", "hook phrase", …). Clip boundaries snap
   to sentence boundaries and scene changes — never cut mid-word.
4. **Auto captions**: per-clip captions from the transcript; editable text,
   timing, and style (font, size, color, highlight color, position, karaoke
   word-highlight on/off); style presets (at least 4).
5. **Smart crop**: per-clip reframe to 9:16, 1:1, 16:9; subject-tracked crop
   window with smoothing; manual override by dragging the crop window.
6. **Timeline editor** (per clip): trim in/out, split, delete segments,
   reorder; caption track; overlay track; audio track with volume; zoom;
   frame-accurate preview.
7. **B-roll**: upload B-roll assets; insert/replace on the overlay track over
   a time range; picture-in-picture or full-frame.
8. **Transitions**: cut, crossfade, slide between segments (FFmpeg xfade).
9. **Sound effects**: upload/choose SFX; place at timestamps; volume per SFX;
   ducking of main audio under SFX optional.
10. **CTA overlays**: text/image overlays ("Follow for more", logo) with
    position, timing, and animation in/out (fade/slide).
11. **Templates**: a saved bundle of caption style + crop ratio + CTA + brand
    colors; apply to any clip; ship ≥3 built-in templates (TikTok bold,
    Shorts clean, Reels minimal).
12. **Export**: render final MP4 (H.264 + AAC) per platform preset — TikTok
    9:16 1080×1920, Shorts 9:16 1080×1920 ≤60 s warning, Reels 9:16, plus
    1:1 and 16:9; quality presets; progress reporting; download link.
13. **Project dashboard**: list projects, processing status, clips grid with
    thumbnails, per-clip export status.

## Non-goals

- No auth/multi-tenancy (single local user)
- No direct posting to social platforms (export files only)
- No cloud storage; local filesystem under `data/`
- No GPU requirement; everything runs CPU-only
